# Markdown 标题序号修复

修复 `.md` 文件中因粘贴其他AI解释内容导致的标题序号冲突和层级混乱问题。

## 触发条件

当用户提到以下关键词时自动触发：
- "标题冲突"、"标题序号"、"排版"、"重新编号"
- "heading"、"标题层级"
- 粘贴AI解释后标题混乱
- 章节序号不连续或重复

## 问题场景

用户会将其他AI的详细解释粘贴到已有的 `.md` 文件中，导致：
1. **序号冲突**：AI解释的标题从"一"重新开始，与原文的"一"重复
2. **层级混乱**：AI解释可能使用 `#` 或 `##`，与原文层级不匹配
3. **子节号冲突**：如 AI 解释用 `4.1`，原文已有 `4.1`
4. **缩进问题**：AI解释内容可能有额外的缩进（2空格或4空格）
5. **对话残留**：AI解释可能包含"你说得对"、"好的，我们继续"等对话内容

## 修复流程

### Step 1: 全面了解文件结构

使用 Grep 获取所有标题结构：
```
pattern: ^#{1,4} 
output_mode: content
-n: true
```

对**每个需要修复的文件**都执行此操作，了解：
- 原始内容的标题结构（通常在文件前半部分）
- AI解释的标题结构（通常在文件中间，序号从"一"重新开始）
- 已添加内容的标题结构（通常在文件末尾）

### Step 2: 识别AI解释块

AI解释块通常有以下特征：
- **对话开头**："你说得对"、"好的，我们继续"、"这是一个很敏锐的问题"
- **重新编号**：标题从"一、二、三..."重新开始
- **额外缩进**：内容行首有2空格或4空格
- **位置**：紧挨在原始知识点后面

### Step 3: 分析冲突

识别以下类型的冲突：
- **一级标题冲突**：`## 一` 出现多次
- **子节号冲突**：如 `### 3.1` 在不同父节下重复
- **层级错误**：`#` 应为 `##`，或 `##` 应为 `###`
- **缩进问题**：内容行首有额外空格

### Step 4: 制定重编号方案

为每个文件制定连续编号方案：

**原则：**
- 原始内容保持不动（通常是 一、二、三...）
- AI解释内容接续原始内容的最后一个编号
- 已添加内容继续接续
- 子节号必须匹配父节号（如父节是"五"，子节应为 `5.1, 5.2, ...`）

**示例：**
```
原文：一、二、三、四
AI解释：一、二、三、四 → 改为 五、六、七、八
已添加：五、六 → 改为 九、十
```

### Step 5: 执行编辑

使用 Edit 工具逐个修改标题：

```
file_path: 目标文件路径
old_string: "## 一、AI解释的标题"
new_string: "## 五、AI解释的标题"
replace_all: false
```

**注意事项：**
- 如果 `old_string` 有多个匹配，需要提供更多上下文来唯一标识
- 子节号也要同步修改（如 `### 1.1` → `### 5.1`）
- 层级错误的标题要同时修正（如 `#` → `##`）

### Step 6: 修复子节号

父节号修改后，子节号必须同步更新。

使用 Grep 查找所有子节：
```
pattern: ^### 
output_mode: content
-n: true
```

**修复规则：**
- 父节 `## 二十、投资回报计算` → 子节应为 `### 20.1, 20.2, ...`
- 父节 `## 十五、怎么求稳态分布` → 子节应为 `### 15.1, 15.2, ...`
- 无编号的子节（如 `### 情况 A`）不需要修改

**常见错误：**
- 父节改了编号，但子节还是旧编号（如 `### 7.1` 应为 `### 20.1`）
- 子节号与父节号不匹配（如父节是"五"但子节是 `3.1`）

### Step 7: 修复缩进问题

AI解释内容可能有额外的缩进，需要移除：

**检测方法：**
```
pattern: ^  [^ ]  # 以2空格开头，但第3个字符不是空格
output_mode: content
-n: true
```

**修复方法：**
- 使用 Edit 工具移除行首的额外空格
- 或者使用正则表达式批量替换

### Step 8: 验证结果

修复完成后，再次使用 Grep 验证：

**验证父节：**
```
pattern: ^## 
output_mode: content
-n: true
```

**验证子节：**
```
pattern: ^### 
output_mode: content
-n: true
```

检查：
- 所有 `##` 级标题序号是否连续
- 子节号是否与父节匹配（如 `## 十五` 下的子节应为 `### 15.x`）
- 是否还有重复序号
- 层级是否正确（`#` vs `##` vs `###`）
- 是否还有额外的缩进

## 常见修复模式

### 模式1：AI解释重置为一~N
```
原文：## 一、xxx → ## 四、xxx
AI解释：## 一、yyy → ## 五、yyy
AI解释：## 二、zzz → ## 六、zzz
```

### 模式2：子节号与父节不匹配
```
原文：## 五、夏普利值
      ### 3.1 问题 → ### 5.1 问题
      ### 3.2 思想 → ### 5.2 思想
```

### 模式3：一级标题需降级
```
原文：# 为什么存在稳态分布 ≠ 收敛？
改为：## 十九、为什么存在稳态分布 ≠ 收敛？
```

### 模式4：无编号标题需添加编号
```
原文：## 贝尔曼方程（Bellman Equation）
改为：## 十三、贝�曼方程（Bellman Equation）
```

### 模式5：AI解释作为子节
```
原文：## 五、回报 (Return) 与折扣因子
      ### 5.1 回报 (Return)
AI解释：## 核心原因：时间越远，折扣越多 → ### 5.1.1 核心原因：时间越远，折扣越多
AI解释：## 一、直观理解 → ### 5.1.2 直观理解
AI解释：## 二、数学解释 → ### 5.1.3 数学解释
```

### 模式6：额外缩进需要移除
```
原文：  ## 一、直观理解  (有2空格缩进)
改为：## 一、直观理解  (移除缩进)
```

## 注意事项

- **绝不改变内容**：只修改标题序号、层级和缩进，不改动任何正文
- **保持连续性**：序号必须连续，不能跳跃
- **子节匹配**：子节号必须与父节号对应
- **批量处理**：一次性处理一个文件的所有冲突，避免中间状态
- **识别AI特征**：注意对话开头、重新编号、额外缩进等特征
- **单行压缩文本预处理**：如果文档由于编辑器转换或粘贴导致大量段落和标题被合并压缩在单一物理行中，必须先将其按句意分段，然后再进行标题序号和层级的排版。
- **排版对齐与空行**：在重新格式化文档结构时，确保各级标题的前后都有空白行，以维持文档的清晰与美观，并保证 Markdown 解析器的正常解析。
- **表格（Table）的空行与格式规范**：所有 Markdown 表格的前后必须各留出一行空行。且尽量避免在单元格内使用 `<br>` 进行换行，改用更简洁的语言描述或分行书写。
